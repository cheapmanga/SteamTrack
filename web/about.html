About steamtrack

What this site records, where it comes from, and how to query it.

What it does

steamtrack watches a hand-picked list of games on Steam and writes down what changes about them. When a game ships a new build, renames a branch, adds or removes a depot, edits its store page, changes price, or posts a patch note, that event is stored and shown here with a timestamp and a before/after view of exactly which fields moved.

It also samples how many people are playing each tracked game, so the player count builds up into a chart over time.

It is not a catalogue of all of Steam. Only games that have been explicitly added are tracked — you can see the full list on the Apps page.

Where the data comes from

Everything comes from Steam itself. The collector connects to Steam as an anonymous user — no account, no login, no credentials — and subscribes to PICS, the same public product-information feed that Steam clients use to learn that an app has been updated. This is the same public source SteamDB reads. Player counts and store details come from Steam's own public endpoints.

Nothing here is scraped from SteamDB, and nothing is guessed. If a field appears on this site, Steam published it.

The one big limitation: history starts when tracking starts

A game's history on this site begins on the day it was added, not on the day the game was released. If a game was added last week, you will see last week's changes and nothing before that.

This is not an oversight, and it is not something that can be fixed by waiting. Steam's PICS feed only announces changes as they happen — it does not let anyone ask for past changelists. In the same way, the number of people playing a game yesterday is not republished anywhere; if nobody was sampling it at the time, that number is simply gone. Backfilling the past is impossible from the live feed alone.

There is one way around it: a SteamDB export for a game can be imported by hand, which pushes that game's history further back. Where that has been done, the older entries you see came from that import rather than from the live feed.

So the honest summary is: the further back you look, the thinner it gets, and the earliest date shown for a game is the date tracking began.

Using the API

Everything the website shows is served by a public JSON API on this same host. Responses are plain JSON, and the interactive reference with every endpoint and parameter lives at /docs.

Authentication and quota

You do not need a key to try it. Requests without a key get an hourly allowance counted per IP address, currently a limited number of requests per hour. Browsing this website draws on that same allowance, since the pages call the public API just like you would.

For heavier or automated use, ask for an API key and send it in the X-API-Key header. A key carries its own hourly budget, independent of the address you call from.

Every answer carries your budget in its headers: X-RateLimit-Limit, X-RateLimit-Remaining and X-RateLimit-Reset (the UTC time your counter goes back to zero). Going over the limit returns HTTP 429; wait for the reset.

Examples

List every tracked game:

curl https://example.org/v1/apps

The same call with an API key:

curl -H "X-API-Key: YOUR_KEY" https://example.org/v1/apps

The ten most recent changes for one game (730 is Counter-Strike 2):

curl "https://example.org/v1/apps/730/changes?limit=10"

The player-count history of a game:

curl https://example.org/v1/apps/730/players

The combined feed of recent changes across every tracked game:

curl "https://example.org/v1/changes?limit=50"

Check what is left of your quota without printing the body:

curl -sS -D - -o /dev/null https://example.org/v1/apps | grep -i x-ratelimit

Main endpoints

EndpointReturns
/healthService status and collector position
/v1/appsEvery tracked game
/v1/apps/{appid}One game and its summary
/v1/apps/{appid}/changesThat game's change history
/v1/apps/{appid}/buildsBuild IDs over time
/v1/apps/{appid}/depotsDepots, branches and manifests
/v1/apps/{appid}/playersPlayer-count samples
/v1/apps/{appid}/pricesPrice history
/v1/apps/{appid}/patchesPatch notes and announcements
/v1/changesAll games, most recent first
/v1/searchFind a tracked game by name

The full list, with every query parameter, is at /docs.

Fair use

Please cache what you fetch and avoid polling in a tight loop. The collector only learns about a change when Steam announces it, so checking more than once every few minutes tells you nothing new. This runs on a small server; the quota exists so that one busy script cannot take the site down for everyone else.
